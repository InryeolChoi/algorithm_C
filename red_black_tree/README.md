# 레드블랙 트리란?
## 🌳 Red-Black Tree 정리

레드-블랙 트리는 자가 균형 이진 탐색 트리(Self-balancing Binary Search Tree) 중 하나임.
대표적으로 Java TreeMap, TreeSet, C++ map, set 내부 구현에 사용됨.

핵심 목적은 트리가 한쪽으로 치우쳐서 성능이 나빠지는 것을 방지하는 것임.

즉,
* 일반 BST → 최악 O(N)
* Red-Black Tree → 항상 O(log N)

⸻

# 1. 기본 개념

Red-Black Tree는 기본적으로 **Binary Search Tree(BST)** 이다.

BST의 규칙 노드에 색깔(red / black) 을 하나 추가한 BST임.

즉, 다음 규칙은 계속 유지됨.

왼쪽 서브트리 < 부모 노드 < 오른쪽 서브트리

```text
    10
   /  \
  5    20
```

하지만 일반 BST는 다음 문제가 있다.

```text
1 → 2 → 3 → 4 → 5
```

이렇게 삽입되면 트리가 **Linked List처럼 변형되어 O(N)** 이 된다.

이를 해결하기 위해 등장한 것이 **Balanced BST**이다.

대표적인 예

- AVL Tree
- Red-Black Tree

---

# 2. Red-Black Tree의 특징

Red-Black Tree는 **각 노드에 색상 정보를 추가한 BST**이다.

예시
```text
    10(B)
   /     \
 5(R)   20(R)
```
여기서

- B = Black
- R = Red


---

# 3. Red-Black Tree의 5가지 규칙

Red-Black Tree는 다음 **5개의 규칙을 항상 만족해야 한다.**

### 1. 모든 노드는 Red 또는 Black이다

각 노드는 반드시 하나의 색을 가진다.


### 2. Root 노드는 항상 Black이다.

### 3. 모든 Leaf(NIL)는 Black이다.

실제 구현에서는 **NULL 노드를 Black으로 간주한다.**

```
    10(B)
   /     \
 5(R)   20(R)
```
각 끝에는 보이지 않는 NIL 노드가 존재한다.

### 4. Red 노드의 자식은 반드시 Black이다

즉 **Red-Red가 연속으로 나타날 수 없다.**

잘못된 예
```
    10(B)
     /
   5(R)
   /
 3(R)
```

올바른 예
```
    10(B)
     /
   5(R)
   /
 3(B)
```

### 5. 모든 경로의 black 노드 수는 같음

루트에서 leaf까지 갈 때 black 노드 개수가 항상 동일

예
```
        10(B)
       /     \
    5(R)     20(R)
   /   \     /   \
 NIL  NIL  NIL   NIL
```

모든 경로
```
10 → 5 → NIL
10 → 20 → NIL
```
black 개수 동일


# 4. 왜 균형이 유지될까?

레드블랙 트리는 완벽한 균형은 아니지만 높이가 항상 제한됨

증명 결과: $\text{height} \le 2\log_{2}{(n+1)}$

즉 $O(\log N)$을 유지함.

# 5. 연산과정
| 연산 | 시간
| --- | ----
| Search | $O(\log N)$
| Insert | $O(\log N)$
| Delete | $O(\log N)$

# 6. 활용사례
* Java : Treemap, TreeSet
* C++ : std::map, std::set
* Linux Kernel : scheduler, memory management

# 7. 삽입 과정 설명

`redblacktree.c`의 삽입은 크게 3단계로 진행된다.

1. `insertNode()` 에서 일반 BST 방식으로 새 노드를 트리에 붙인다.
2. 새 노드를 `RED` 로 설정하고, 왼쪽/오른쪽 자식을 `Nil` 로 연결한다.
3. `rebuildAfterInsert()` 에서 레드-블랙 트리 규칙이 깨졌는지 검사하고, 필요하면 재색칠(recoloring)과 회전(rotation)을 수행한다.

## 1. insertNode(): 삽입 시작

`insertNode()` 는 삽입의 메인 함수이다.

- 먼저 `insertNodeHelper()` 를 호출해서 새 노드를 BST 규칙에 맞는 위치에 삽입한다.
- 그 다음 새 노드의 색을 `RED` 로 바꾼다.
- 마지막으로 `rebuildAfterInsert()` 를 호출해 균형을 다시 맞춘다.

새 노드를 처음부터 `BLACK` 이 아니라 `RED` 로 넣는 이유는, black 높이(각 경로의 black 노드 수)를 바로 증가시키지 않기 위해서이다.  
대신 red-red 충돌이 생길 수 있으므로, 그 부분만 후처리로 해결한다.

## 2. insertNodeHelper(): BST 방식으로 위치 찾기

`insertNodeHelper()` 는 값 비교를 통해 삽입 위치를 찾는다.

- 현재 노드보다 값이 크면 오른쪽으로 이동한다.
- 현재 노드보다 값이 작으면 왼쪽으로 이동한다.
- 자식이 `Nil` 인 위치를 만나면 그 자리에 새 노드를 연결한다.

즉 이 단계에서는 **색 균형은 신경 쓰지 않고**, 먼저 이진 탐색 트리 구조만 만족하도록 삽입한다.

예를 들어 `10`, `20`, `15` 순서로 넣으면:

```text
10
  \
   20
  /
15
```

처럼 BST 위치에 먼저 들어간 뒤, 이후 단계에서 색과 회전을 조정한다.

## 3. rebuildAfterInsert(): 규칙 복구

`rebuildAfterInsert()` 는 새로 삽입된 노드 `X` 를 기준으로 위쪽으로 올라가며 규칙을 복구한다.

반복 조건은 다음과 같다.

- `X` 가 루트가 아니고
- `X` 의 부모가 `RED` 인 동안

즉, **red-red 위반이 발생한 경우에만** 복구 작업을 수행한다.

### 경우 1. 부모와 삼촌이 모두 RED

코드에서 부모가 할아버지의 왼쪽 자식인 상황에서:

- 부모(`parent`)가 `RED`
- 삼촌(`uncle`)도 `RED`

이면 회전하지 않고 색만 바꾼다.

- 부모를 `BLACK`
- 삼촌을 `BLACK`
- 할아버지를 `RED`

로 바꾼 뒤, 검사 기준 노드 `X` 를 할아버지로 올려서 다시 확인한다.

이 과정은 아래처럼 이해할 수 있다.

```text
      G(B)
     /   \
   P(R) U(R)
   /
 X(R)
```

재색칠 후:

```text
      G(R)
     /   \
   P(B) U(B)
   /
 X(R)
```

현재 서브트리의 red-red 문제는 해결되지만, `G` 가 위쪽 부모와 다시 red-red 가 될 수 있어서 반복문을 계속 돌게 된다.

### 경우 2. 삼촌은 BLACK, X가 안쪽 자식인 경우

부모는 빨간색인데 삼촌이 `BLACK` 이고, `X` 가 꺾인 모양이면 먼저 회전으로 일직선 형태를 만든다.

예를 들어 부모가 할아버지의 왼쪽 자식인데 `X` 가 부모의 오른쪽 자식이면:

```text
    G
   /
 P
  \
   X
```

이 경우 코드에서는:

- `X = X->parent`
- `rotateLeft(root, X)`

를 수행한다.

즉, 먼저 작은 회전으로 구조를 바꿔 다음 단계의 회전을 적용하기 쉬운 형태로 만든다.

### 경우 3. 삼촌은 BLACK, X가 바깥쪽 자식인 경우

위의 정리 이후에는 일직선 형태가 되므로, 부모와 할아버지의 색을 바꾸고 할아버지를 기준으로 회전한다.

- 부모를 `BLACK`
- 할아버지를 `RED`
- `rotateRight(root, grandparent)`

형태로 처리한다.

예시는 다음과 같다.

```text
      G(B)
     /
   P(R)
   /
 X(R)
```

처리 후:

```text
      P(B)
     /   \
   X(R)  G(R)
```

이렇게 하면 red-red 위반이 사라지고, 트리 높이도 한쪽으로 치우치지 않도록 조정된다.

## 4. rotateLeft(), rotateRight() 의 역할

회전 함수는 부모-자식 관계를 다시 연결해서 서브트리의 모양만 바꾸고, BST의 중위 순서 자체는 유지한다.

- `rotateLeft()` : 오른쪽으로 치우친 구조를 왼쪽으로 당겨 균형을 맞춘다.
- `rotateRight()` : 왼쪽으로 치우친 구조를 오른쪽으로 당겨 균형을 맞춘다.

코드 주석대로 회전 중에는 다음 연결을 다시 잡는다.

- 회전 축이 되는 부모와 자식 연결
- 서브트리(`T2`) 연결
- 조부모와 새 서브트리 루트 연결

즉, 회전은 단순히 방향만 바꾸는 것이 아니라 **부모 포인터와 자식 포인터를 모두 다시 맞추는 작업**이다.

## 5. 삽입 전체 흐름 요약

`redblacktree.c` 기준 삽입 순서는 아래와 같이 볼 수 있다.

```text
새 노드 생성
-> BST 방식으로 삽입
-> 새 노드를 RED 로 설정
-> red-red 위반 검사
-> 필요하면 recoloring
-> 필요하면 rotation
-> 루트까지 규칙이 만족될 때까지 반복
```

결국 레드-블랙 트리의 삽입은  
**"일단 BST처럼 넣고, 이후에 색 변경과 회전으로 규칙을 복구하는 과정"** 으로 이해하면 된다.
